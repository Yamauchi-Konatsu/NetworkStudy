ワイヤレス
・WLC（wireless LAN controller） ：オフィスや学校などで多数の無線LANアクセスポイントを利用する際に複数のアクセスポイントを統合管理する装置
・集中管理型AP：管理者が設定する対象が無線LANコントローラ（APは電波の送受信のみ、設定や制御はWLC)
・自律型AP：管理者が設定する対象がアクセスポイント（AP単独でWLANを提供でき、各APがそれぞれ設定、制御を行う)　WLCがなくてもWLANを構築できるアクセスポイント
・CSMA/CA：無線LANを用いた通信の衝突を回避する方法
・SOHO（small office home office): 小規模なネットワーク構成
・BSS：１つのAPとそのAPの電波内にいる配下の無線LANクライアントで構成されるネットワーク
・BSSID：48ビットの数値
・ESS：複数のBSSで構成される無線LAN
・ESSID：最大32文字までの英数字を設定できる
・ローミング：無線LANで移動に応じて使用するアクセスポイントを切り替える
・CAPWAP：APとWLCの間のWLANクライアントトラフィックのカプセル化と転送に加えて、APとWLANの設定や管理を行う

FHRP
・HSRP：Cisco独自、デフォルトゲートウェイを冗長化するための仕組み
・VRRP：標準化、デフォルトゲートウェイを冗長化するための仕組み
・GLBP：Cisco独自、ひとつのグループ内で利用可能なデフォルトゲートウェイを複数存在させることができデフォルトゲートウェイの負荷分散ができる
・アクティブルータ：実際に転送処理しているルータ
・スタンバイルータ：アクティブルータが通信できなくなった時のためのバックアップとなるルータ

QoS
・輻輳管理：パケットに割り当てられた優先順位を元に送信する順番を決定することで輻輳をコントロールできること
・輻輳回避 : 輻輳によって起こるテールドロップを防ぐためにキューに待機しているパケットを破棄すること
・テールドロップ : キューがいっぱいになったときに、それ以降に届いたパケットはすべて破棄してしまうこと
・帯域制御 : 特定の基準を元に使用できるデータを転送量や通信速度を制限すること
・シェーピング : データを送信する側で決められた制限レートを超えるパケットは一旦キューにおいて送信可能になったタイミングで送信するもの
・ポリシング : ネットワークの入り口部分で制限レートを超えるパケットは破棄される

WAN
・VPN : インターネット上に特定の人だけが使える仮想の専用回線を構築する
・インターネットVPN : インターネット回線を使う
・サイト間VPN : 企業や支社など２つ以上のネットワーク間の接続＝拠点同士の接続
・リモートアクセスVPN : 暗号化されたIPsecを介してリモートワーカーの認証と企業のデータセンターやクラウドにあるアプリケーションおよびデータへのアクセスを可能にするネットワークセキュリティ技術＝拠点とデバイスの接続
・IPsec : インターネットプロトコルにセキュリティを構築すること(認証→暗号化→トンネリング)マルチキャスト、ダイナミックプロトコルが使えない、データが暗号化
・GRE : マルチキャスト、ダイナミックプロトコルが使える、暗号化されない。トンネリングプロトコルの一種。　
・クラウド : インターネットなどのネットワーク経由でユーザーにサービスを提供する形態のこと
・SaaS：ソフトウェアをインターネット越しに使わせてくれるサービス
・PaaS：サーバやOS、データベースなどプラットフォームをインタネット越しに使えるようにしてくれるサービス
・IaaS：サーバや回線などインフラに分類される部分をインターネット越しに使えるようにしてくれるサービス

セキュリティ
・脆弱性：脅威によって利用されるおそれのある弱点
・脅威：情報システムに危害を与えるセキュリティ事故の潜在的な原因。人為的なものと環境的なものがある。
・マルウェア : コンピュータやその利用者に被害をもたらすことを目的とした悪意のあるソフトウェア
・ウイルス : 単体で存在できず、宿主を必要とし、自己増殖する
・ワーム : 単体で存在を確立し、自己増殖する
・トロイの木馬 : なりすまして存在、自己増殖しない
・スプーフィング : 第三者が正規の人物になりすましコンピュータネットワークを不正に利用するサイバー攻撃のこと
・Dos攻撃 : アクセスが集中してサーバがパンクすることを利用し、一つのコンピュータが悪意を持ってサーバに大量のデータを送りつけるサイバー攻撃
・DDoS攻撃 : 複数のコンピュータから一斉にサイバー攻撃をしてくること
・中間者攻撃 : 第三者が二者同士の通信へ不正に割り込み通信データの盗聴や改ざんを行うサイバー攻撃
・ソーシャルエンジニアリング : ネットワークに侵入するために必要なパスワードなど重要な情報を、情報通信技術を使用せずに盗み出す方法
・ブルートフォースアタック：片っ端から総当たりで試していく、パスワードや暗号の解読方法
・WPA : 無線LAN上で通信を暗号化して保護する
・WPAパーソナル : 暗号化の規格がWPAで事前共有鍵（PSK）を使って認証するやり方
・WPAエンタープライズ : 暗号化する際の規格がWPAで認証サーバを使って認証するやり方
・サプリカント : ネットワーク上の利用者や端末の認証において、認証を要求する側（クライアント側）の機器やソフトウェアのこと
・オーセンティケータ : ユーザーの認証情報を扱うデバイスのこと
・認証サーバ : 認証関連のサービスを提供するコンピュータ
・多要素認証(MFA) : ①本人のみ知っていること(パスワード)②本人のみ持っている物(ｉｄカード)③本人の特徴(指紋)のうち、2つ以上の要素を確認することで行う認証のやり方
・知識要素 : パスワードや秘密の質問など、ユーザー自身が記憶している情報を使って行われる認証方法
・所持要素 : 本人のみが所持している物であり、スマートフォン、ICカード、スマートカード、ワンタイムパスワードのトークンなど
・生体要素 : ユーザーの体の特徴を活用して認証を行う
・AAA : システムに接続するユーザが正しいかを確認し、そのユーザにアクセス許可を与え、ユーザの接続時間や操作などを記録する機能を提供する枠組み
・Authentication : 認証
・Authorization : 認可。認証に成功したユーザに対して提供する権限を制限する
・Accounting : アカウティング。認証に成功したユーザに対して情報を収集する
・RADIUS（Remote Authentication Dial-In User Service）：認証するときに使うプロトコル
